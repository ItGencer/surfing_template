# Project Context

Оновлено: 2026-07-03.

## Коротко

`Surfing_template` - фронтенд-шаблон сайту "Surfing Institute". Це промо/landing page для surf-школи або інституту серфінгу. Основний досвід - візуальний, з великими фото хвиль і серферів, спортивним темпом, чіткою секційною структурою і сильним оранжевим брендовим акцентом.

## Користувачі

- Потенційні студенти surf-школи: початківці, діти й підлітки, туристи, intermediate/advanced surfers.
- Власник або команда surf-школи, які хочуть презентувати курси, локації, інструкторів і довіру.
- Розробник або AI-білдер, який має відтворити дизайн у коді, Figma або іншому інструменті.

## Цілі

- Відтворити сайт "Surfing Institute" як сучасний, енергійний, спортивний landing.
- Показати ключові секції: Header, Hero, Marquee, About, Courses, Locations, Process, Gallery, Testimonials, FAQ, Footer.
- Зберегти фірмові патерни: full-bleed фото, оранжевий ticker, `SURF*ING`, scribble underline, overlay-картки курсів.
- Забезпечити адаптивність і ручну перевірку основних взаємодій.

## Нецілі

- Бекенд, база даних, авторизація, платежі, реальне бронювання уроків - не підтверджені.
- Реальна інтеграція форми Contact/Enroll - `to confirm`.
- Аналітика, SEO-автоматизація, CMS, мультимова підтримка - `to confirm`.
- Підключення зовнішніх платних фото, шрифтів або сервісів без явного підтвердження.

## Підтверджений стек

З `package.json` і `webpack.config.js` підтверджено:

- Webpack 5.
- Vanilla JavaScript entry: `src/index.js`.
- HTML template: `src/index.html`.
- Sass підтримка через `sass-loader`.
- Production output: `public/`.
- Dev server: порт `3000`.

`to confirm`:

- Чи створювати `src/styles.scss` як головний stylesheet.
- Чи додавати `src/assets/` для локальних зображень.
- Чи використовувати локальні шрифти, Google Fonts або системні fallback-шрифти.
- Чи перейменовувати npm package з `my-portfolio-card`.

## Поточний стан реалізації

- `src/index.html` містить базову HTML-структуру і title `Surf * iing`.
- `src/index.js` порожній.
- Готової верстки, стилів, зображень і інтерактивності поки немає.
- Ця документація створена як основа для наступної реалізації.

## Дизайн-напрям

- Тема: surf-школа / surfing institute.
- Настрій: енергійний, сучасний, спортивний.
- Кольори: білий/чорний контраст + яскравий оранжевий акцент.
- Макет: центральний контейнер `~1200-1280px`, великі вертикальні відступи `~96-120px`.
- Фото: багато full-bleed зображень хвиль, океану і серферів.
- UI: плаский, мінімум тіней, картки зі світло-сірим фоном, pill-кнопки.

## Архітектурні орієнтири

- Односторінковий landing без routing.
- HTML має залишатися семантичним: `header`, `main`, `section`, `footer`, списки для навігації і FAQ.
- Повторювані UI-патерни бажано оформити як HTML/CSS класи і невеликі JS-хелпери:
  - section header;
  - marquee ticker;
  - course card;
  - location card;
  - stat card;
  - process step card;
  - testimonial card;
  - FAQ item.
- JavaScript потрібен тільки для поведінки: scroll progress, accordion, mobile menu, можливо плавний scroll.

## Обмеження

- Не вигадувати реальні контакти, адреси, ціни, URL, API, secret keys.
- Якщо потрібен контент, використовувати placeholder-тексти з дизайн-опису або позначати як `to confirm`.
- Не використовувати темні, розмиті, надто обрізані фото там, де користувач має бачити серфінг/локацію.
- Не перетворювати сторінку на маркетингову hero-only заглушку: перший екран має бути реальним hero сайту, а нижче - повна структура landing.
